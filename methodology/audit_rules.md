# 报告审计规范

每次生成报告后、交付前必须执行以下审计。

---

## 1. 数据溯源审计

报告中的每个关键数字必须能追溯到数据库。

**审计方法**：运行独立查询脚本，复算以下指标并与报告比对：
- 总样本量、有效薪资数量
- 整体中位数、P25、P75、P90
- 高薪岗位数量和占比
- 各方向岗位数、中位数、P75、P90、高薪占比
- 经验×方向交叉样本量

**允许差异**：因爬虫持续运行导致的微小差异（<5%）需标注。

---

## 2. 数字一致性审计

同一指标在不同章节必须前后一致。

**重点检查**：
- 总岗位数在第0章、第1章是否一致
- 高薪岗位数在第0章、第4章是否一致
- 各方向岗位数在第3章、第7章是否一致
- 正文描述和表格数字是否一致

---

## 3. 口径一致性审计

不得混用不同统计口径：
- "岗位数"始终指有效薪资岗位数
- "高薪"始终指月薪中位数 ≥ P75
- "溢价"始终以整体中位数为基准
- "方向分类"始终使用 title+skills+description 重新聚类

---

## 4. 结论支撑审计

每个结论必须有数据支撑：
- 数据事实：可直接复算
- 数据解读：基于数据的解释
- 求职判断：面向求职者的建议（必须标注为建议性质）

**禁止**把"求职判断"写成"数据事实"。

---

## 5. 小样本警告规则

| 样本量 | 标注 | 允许的表述 |
|--------|------|-----------|
| ≥100 | 高可信 | 可作为稳定结论 |
| 50-99 | 中等可信 | 可参考但需说明样本量 |
| 20-49 | 谨慎 | 只能作为趋势观察 |
| <20 | 小样本 | 不宜单独下结论 |

小样本方向（<50）不得使用"适合主投""岗位充足"等表述。

---

## 6. 通用报告边界规则

通用市场分析部分（第0-10章）：
- 不得出现"你""我""我的背景"等个人化表达
- 不得出现特定候选人的背景描述
- 使用通用表达："有XX经验的候选人"

---

## 7. HTML 与 Markdown 一致性

两个版本信息内容必须完全一致：
- 章节结构一致
- 数据表格一致
- 结论文字一致

每次修改必须同时更新 `generate_html()` 和 `generate_markdown()`。

---

## 8. 脱敏版额外检查

公开版（脱敏版）必须确认：
- 无具体平台名（Boss直聘等）
- 无真实公司榜单
- 无单条岗位精确薪资
- 无"岗位+公司+薪资+来源"组合
- 无招聘者信息
- 无可反查的岗位标题
- 有免责声明

---

## 审计脚本模板

```python
import re
from models.database import init_db, get_engine
from models.tables import JobRecordDB
from sqlalchemy.orm import Session
from sqlalchemy import func
import numpy as np

# 独立复算
init_db()
engine = get_engine()
with Session(engine) as s:
    total = s.query(JobRecordDB).count()
    # 有效样本（salary_mid < 100K）
    valid = s.query(JobRecordDB).filter(
        JobRecordDB.salary_min.isnot(None),
        (JobRecordDB.salary_min + JobRecordDB.salary_max) / 2 < 100
    ).count()
    print(f"总记录: {total}, 有效样本: {valid}")
    # ... 继续复算各指标
```
