# 分析原则与方法论

## 分析目标

帮助求职者做真实求职判断，回答：
1. 哪些方向高薪岗位密集？
2. 哪些方向看起来热门但薪资一般？
3. 高薪 JD 和普通 JD 的能力差异在哪？
4. 哪些标签只是方向标签，哪些是能力信号？
5. 经验在哪些方向溢价最明显？

---

## 数据清洗规则

### 薪资标准化
- 支持格式：15-25K、15-25K·14薪、1.5-2.5万、面议
- 统一为：salary_min（月薪下限K）、salary_max（月薪上限K）、salary_months
- 薪资中位数 = (salary_min + salary_max) / 2
- "面议"和无法识别的格式标记为 None

### 极端值处理
- 月薪中位数≥100K 的岗位不参与统计计算
- 仅在"极端高薪画像"中展示
- 这些岗位多为猎头发布，薪资范围虚高

### 方向重新聚类
- 不依赖平台原始分类
- 基于 title + skills + description 关键词匹配
- 多个方向，优先级从上到下（见 `analysis/analysis_method_example.py` 中的聚类规则）

## 分析维度

### 核心维度
| 维度 | 指标 | 用途 |
|------|------|------|
| 薪资分层 | 中位数、P25、P75、P90 | 判断整体水平和上限 |
| 高薪占比 | P75+岗位占比 | 判断高薪机会密度 |
| 溢价 | (方向中位数-整体中位数)/整体中位数 | 判断方向薪资优势 |
| P75溢价 | (方向P75-整体P75)/整体P75 | 判断高薪段优势 |
| 样本可信度 | ≥100高可信，50-99中等，20-49谨慎，<20小样本 | 判断结论稳定性 |

### 交叉分析
- 经验 × 方向（经验溢价）
- 行业 × 薪资
- 公司类型 × 高薪占比
- 技能标签 × 薪资

### 能力因子分析
高薪 JD vs 普通 JD 的核心差异：
1. 业务闭环（对结果负责）
2. 指标驱动（数据决策）
3. 策略设计（设计策略并验证效果）
4. AI 应用落地
5. 复杂系统抽象
6. Owner/负责人能力

## 结论表达规范

### 必须区分三类表述
- **数据事实**：可从数据直接复算（如"AI方向有1586个岗位"）
- **数据解读**：基于数据的解释（如"AI方向高薪岗位密集"）
- **求职判断**：面向求职者的建议（如"AI方向适合重点投递"）

### 禁止的表述
- "薪资溢价最高"（当中位数溢价为0%时）
- "整体薪资更高"（当中位数与整体持平时）
- "高薪必备技能"（标签≠能力，相关≠因果）
- "建议提升能力"（没说补什么、怎么补）

### 正确的表述
- "高薪岗位密集"（基于高薪占比）
- "P75/P90 有正溢价"（基于具体数据）
- "高薪 JD 中更常见的标签"（不暗示因果）
- "补齐XX能力（2-3周可入门），具体方式：..."

### 标签 vs 能力
- 方向标签（社交产品、用增产品）：反映岗位方向，不等同于技能
- 能力信号（AIGC、搜索产品、大模型）：可作为简历方向参考
- 小样本趋势词（样本<20）：仅作观察
- 无区分度标签：普遍出现，无法拉开差距

## 方向分类规则

第7章方向判断的分类逻辑：
1. **稳定高薪方向**：高薪占比≥35% 且 样本≥100，或 P75有正溢价 且 样本≥50
2. **高薪机会多但无中位数溢价**：高薪占比≥28% 且 中位数溢价≥0 且 P75溢价≤0 且 样本≥100
3. **天花板高但需实力**：P90溢价>0 且 中位数溢价<0 且 样本≥20
4. **小样本高上限**：样本<50 且 高薪占比>30%
5. **岗位多但薪资一般**：样本≥50 且 中位数溢价<0
6. **不建议主攻**：溢价<-20% 或 样本<20
